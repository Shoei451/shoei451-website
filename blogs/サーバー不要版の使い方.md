## 📝 新しい記事の書き方

### 手順

1. **`posts-list-inline.js` を開く**

2. **`markdownPosts` オブジェクトに追加:**

```javascript
const markdownPosts = {
    "2024-12-26-markdown-sample": { ... },  // 既存
    
    // ← ここに新しい記事を追加
    "2024-12-27-my-new-post": {
        metadata: {
            title: "新しい記事のタイトル",
            titleEN: "My New Post",
            date: "2024-12-27",
            category: "Personal",
            categoryJP: "日記"
        },
        content: `
# 記事のタイトル

ここに**Markdown**で内容を書きます。

## セクション1

内容...

### サブセクション

詳細...

## まとめ

まとめ...
        `
    }
};
```

3. **`blogPosts` 配列にも追加:**

```javascript
const blogPosts = [
    // ← ここに追加（新しい記事ほど上）
    {
        title: "新しい記事のタイトル",
        titleEN: "My New Post",
        date: "2024-12-27",
        category: "Personal",
        categoryJP: "日記",
        excerpt: "記事の簡単な説明...",
        link: "markdown-blog-inline.html?post=2024-12-27-my-new-post",
        image: "../images/favicon.png"
    },
    // 既存の記事...
];
```

4. **保存して、ブラウザで確認！**

## 💡 メリット・デメリット

### ✅ メリット
- **サーバー不要** - ダブルクリックで開ける
- **簡単** - ファイル1つで完結

## 🔄 将来的にサーバー版に移行する場合

サーバー版（markdown-blog.html）に移行する方法:

1. **Markdown内容をファイルに抽出:**
   - `posts-list-inline.js` のcontent部分をコピー
   - `posts/2024-12-27-my-new-post.md` として保存
   - frontmatterを先頭に追加

2. **ローカルサーバーを起動:**
   - `start-server.bat` を使う
   - または VS Code の Live Server

3. **リンクを変更:**
   - `markdown-blog-inline.html?post=...` → `markdown-blog.html?post=...`

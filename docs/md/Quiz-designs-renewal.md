## 問題
- 複数クイズファイル間でのcss変数、element id の不統一による管理コストの高さ
- 新規ファイル制作の際に使える資源が不十分
## 対象ファイル
- history/world/index.html
- history/china/index.html
- seikei/timeline/quiz.html
- others/idiom_quiz/index.html
- 今後制作するファイル
### 備考
1. 地理クイズ2種類（geography/ 内）など、common.cssのheaderなどのコンポーネントを流用しているような小規模クイズは、今回は改修対象としない
2. クイズ以外でも、seikei/timeline/index.htmlとhistory/china/timeline.htmlなど、年表系のファイルは共通化可能（特にこの2つは、カラースキーマと実際に読み込むデータが異なるのみ）
3. あまり関係ないメモ：sub-indexへのbacklink系のリンクが、npm run check で引っかかる（slug指定のため）ため、除外リストへの登録が必要
## 共通範囲の確認
クイズ系のファイルに共通するコンポーネントを列挙。
### スタート画面（設定画面）
- クイズタイトル
- ちょっとした説明（タイトル下）
- テーマ画像(角丸)
- チュートリアル：Markdownをパースし、ハンバーガーボタンを通して開閉式サイドバーで確認できるようにする（ハンバーガーボタンは画面左上に浮遊させる？ position: fixed ; →今のテーマトグルと同じようなイメージ）
- 範囲選択（選択ボタンはseikei/timeline/quiz.htmlと、history/china/index.htmlに共通しているものを流用可能：ボタンはjsで動的生成。ただし、範囲を複数選択可能なタイプのクイズと、範囲が1つのみ選択できるクイズ（history/world/index.html、others/idiom_quiz/index.htmlなど。こちらは似たようなデザインでのスイッチトグルの実装を検討）の2パターンのUIが必要）
- 出題数選択：スライダー+その右側に入力フォーム（半角数字のみ受け付け）

## 方針
- 1機能1js/css：完全に部品化する。こうすることで、4択系と入力系で無駄のない運用ができると考える。quiz-components/フォルダを作るのも検討
- カスタマイズ性能：将来の新規ファイル作成などに備える。config用のファイルを準備する方向。
